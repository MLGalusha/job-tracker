That kind of engineering, where correctness at scale is the entire point, is what I understand financial software requires.

My largest project is Voices That Remain, a 58,700-line TypeScript application with a React 19 frontend, Express backend, and Postgres database. I designed the relational data models, wrote the queries, and built a 4-stage data processing pipeline with structured validation at each stage. The project is live at voicesthatremain.com with 547 commits in the repository. I also reverse-engineered 92 undocumented API endpoints for Staffclaw, a scheduling platform rebuild, which required understanding complex data relationships without any reference documentation.

I do not have a finance background. What I do have is a track record of learning unfamiliar domains fast and building correct systems within them. The ability to write precise, data-integrity-focused software is harder to teach.
