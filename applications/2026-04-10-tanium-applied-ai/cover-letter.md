Applied AI Engineer II

I build products powered by LLMs, and I build them by directing AI coding agents as my primary tools. That combination, shipping AI features while also working through AI daily, is what drew me to this role.

One thing I notice about Tanium's position in the endpoint management space: you have real-time visibility into millions of endpoints, which means your AI agents don't have the data access problem most companies face. The hard part for most teams building AI agents is getting structured, current data to work with. Tanium already solved that with the linear-chain architecture. That changes what's possible with AI on top of it. Instead of spending cycles on data pipelines and retrieval, the interesting engineering is in what the agents actually do with that data, how they reason about endpoint state, surface anomalies, and take action. That's the kind of applied AI work I want to do.

My closest analog to this role is Voices That Remain, a digital archive I built and deployed solo. It has a 4-stage AI pipeline inside it: GPT-5.4 Vision for transcribing handwritten letters, structured outputs with Zod validation for metadata extraction, tool calling for entity resolution across thousands of documents, and cost-tiered model selection so I'm not burning tokens on audit steps. The pipeline runs on GCP with Cloud Run, Cloud SQL, and Cloud Build CI/CD. I designed a human verification gate between transcription and downstream extraction because I wanted zero hallucinated metadata reaching the public archive. That gate is an explicit architectural choice about when to trust the model and when not to.

Before I started working with coding agents, I wrote SonicGen, a 14,000 line Python audio fingerprinting engine, entirely by hand. No agents, no autocomplete. I did that deliberately because I wanted to understand the code before I let agents write it. When I switched to agent-directed development in 2026, the transition was easy because I already knew what the agents were producing. I can debug when the model is wrong.

I don't have a CS degree. I have a 6-month AI bootcamp from UNC Chapel Hill, four shipped projects, 58,700 lines of production TypeScript, and a live site at voicesthatremain.com. I'm 25 minutes from your Durham office.
